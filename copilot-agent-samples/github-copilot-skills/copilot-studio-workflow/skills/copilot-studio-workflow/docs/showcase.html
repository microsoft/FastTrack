1 · The Problem

Copilot Studio is powerful, but the default workflow is still portal-first.

Without an engineering workflow, every change becomes harder to review, harder to package, and harder to trust.

Teams can absolutely ship serious agents with Copilot Studio. The friction comes from the process around it: too much portal state, too little source-control discipline, and too many rules that are only learned after something breaks.

Portal-heavy editing

Important agent changes still feel like point-and-click project work instead of clean, reviewable software development.

No default ALM mindset

Source control, repeatable packaging, and CI/CD habits are possible, but they are not the path most builders start with.

Undocumented platform gotchas

Pulls dirty workflow files, pushes miss solution membership, imports disable flows, and helpers often learn that too late.

Environment sharing is fragile

Moving an agent between environments can leak live URLs, break packaging, or produce artifacts that do not reflect the intended source.

AI assistants lack platform context

Generic coding help does not know the Copilot Studio sync model, Power Platform solution rules, or the PowerClaw team’s proven sequence.

Manual recovery wastes hours

When the loop is unclear, teams debug repo drift, packaging gaps, and import failures instead of building better agent behavior.

6 · Platform Gotchas

The six issues most likely to burn time if your assistant does not know them.

The skill encodes eleven total gotchas. These six are the ones builders hit first and hardest.

OnConversationStart is unreliable

Use OnActivity(type: Message) with an IsBlank() guard for just-in-time initialization across real channels.

Push ≠ solution component

A new topic can exist in Dataverse and still be missing from your exported solution until you add it explicitly.

Global variables need YAML files

UI references are not enough. If the variable definition is missing from variables/, packaging can break.

Cloud pull brings environment URLs

Workflow JSON and settings can come back dirty with live values. Revert immediately unless you intentionally need them.

Solution import deactivates flows

Successful import does not mean operational automation. Reconnect and re-enable flows as part of the checklist.

Extension sync is all-or-nothing

There is no selective push or pull. Coordinate edits, pull first, and avoid overlapping cloud changes.
